===============================================================================

　AccessLamp_D

===============================================================================


■ソフトの概要

　ストレージ毎のアクセス状況 (アクセスランプ) を表示します。


■動作環境

　Windows 10 Pro または Windows 10 Home


■インストール方法

　アーカイブの中身をローカルディスク上の任意の場所にコピーして下さい。


■アンインストール方法

　レジストリなどは一切使っていません。
　ファイルを削除するだけでアンインストールできます。


■起動方法

　AccessLamp_D.exe を実行して下さい。

　初回起動時は画面の左上 (画面の座標 (0, 0) の位置) にウィンドウが表示されます。
　小さくて見つけにくいのでご注意下さい。

　ウィンドウをドラッグして画面上を移動できます。


■終了方法

　ウィンドウを右クリックして「終了」を選択して下さい。

　タスクバーにアイコンを表示している場合は、当該アイコンを右クリックして「ウィンドウを閉じる」を選択しても
　プログラムを終了できます。

　タスクトレイにアイコンを表示している場合は、当該アイコンを右クリックして「終了」を選択しても
　プログラムを終了できます。


■設定

　ウィンドウを右クリックして「設定」を選択して下さい。

　タスクトレイにアイコンを表示している場合は、当該アイコンを右クリックして「設定」を選択しても
　設定を開くことができます。


　●インスタンスの一覧

　　アクセス状態 (アクセスランプ) を表示するインスタンス (ストレージ) を設定します。
　　[表示]と設定された項目のみアクセスランプを表示します。
　　項目の並びは表示するアクセスランプの並びと一致します。
　　アクセスランプ内に描画される文字列は「表示名」です。

　　Up を押下すると、選択されている項目を上へ移動します。

　　Down を押下すると、選択されている項目を下へ移動します。

　　Edit を押下すると、選択されている項目について編集画面 (インスタンス編集) を開きます。

　　一覧を右クリックするとメニューが表示されます。

　　　選択解除   ... 一覧を未選択状態にします。(特に意味はありません)
　　　編集       ... 選択されている項目について編集画面 (インスタンス編集) を開きます。
　　　表示       ... 選択されている項目を [表示] に設定します。
　　　非表示     ... 選択されている項目を [非表示] に設定します。


　●色とフォント

　　余白の色 (背景色)           ... アクセスランプの周囲 (余白の部分) の色を設定します。
　　Denied 背景色 / 文字色      ... アクセス状態を監視できなかった場合の色を設定します。
　　Idle 背景色 / 文字色        ... アイドル状態の色を設定します。
　　Busy 背景色 / 文字色        ... 読み書き中の色を設定します。
　　Very_Busy 背景色 / 文字色   ... 読み書き中 (ビジー状態) の色を設定します。
　　フォント                    ... アクセスランプ内に描画される文字列のフォントを設定します。

　　★フォントサイズを小さくしすぎると、
　　　アクセスランプを視認できなくなる程小さくなってしまう場合があります。ご注意下さい。


　●その他

　　余白の幅                           ... アクセスランプの周囲 (余白の部分) の幅を設定します。単位はピクセル
　　タスクバーにアイコンを表示する     ... タスクバーに当ソフトのアイコンを表示します。
　　タスクトレイにアイコンを表示する   ... タスクトレイに当ソフトのアイコンを表示します。
　　常に手前に表示する                 ... 当ソフトのウィンドウを常に手前に表示します。
　　位置を固定する                     ... 当ソフトのウィンドウをドラッグできないようにします。
　　ランプを縦に並べる                 ... アクセスランプを縦に並べます。

　　切断されたランプの再接続を試みる   ... ストレージが取り外されるなどして2秒程度モニタ不可の状態が続くと
　　                                       アクセスランプは切断状態 (Denied) になり、以後情報取得を行わなくなります。
　　                                       この項目にチェックを入れると、30秒毎にモニタの再接続を試みるようになり、
　　                                       成功した場合はアクセスランプが復活します。
